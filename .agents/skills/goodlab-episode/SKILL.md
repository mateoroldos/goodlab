# Goodlab Episode

An episode is one idea made felt, named, and visualized.

## Steps

1. Name the idea.
   - State the single conceptual move.
   - Completion criterion: the episode has one takeaway; if there are two, split it or ask.

2. Open the loop.
   - Start from a felt problem: confusing code, unexpected behavior, missing intuition, or a real tradeoff.
   - Completion criterion: the viewer has a reason to care before the abstraction appears.

3. Build from examples.
   - Lead with concrete cases, then name the abstraction.
   - Completion criterion: every abstraction is earned by an example already shown.

4. Keep blocks tight.
   - Each text block resolves one small question and creates the next.
   - Completion criterion: no block introduces more than one new concept.

5. Pair with visuals.
   - Use `goodlab-visual` when creating or changing the visual model.
   - Completion criterion: text and visual reinforce the same idea at each step.

6. Ask before wiring.
   - Before adding files to `series.ts`, `catalog.ts`, or route-visible content, ask whether the user wants implementation or just the episode draft.
   - Completion criterion: the episode is either drafted only or fully wired according to the user's answer.

7. Subtract.
   - Remove a weak block, example, or visual state if understanding survives without it.
   - Completion criterion: every remaining piece helps the idea click.
